_init_session(session) — COLD RESET ⚠

▼
proposed
FIX: never cold-reset mid-utterance
Keep one continuous never-finalized context; on a (server-side) speculative finalize, fork a deep copy of the six rolling values + accumulated_audio, run the padded keep_all_outputs flush on the fork, and discard it if speech resumes within the debounce window. The live context is never touched → reproduces the ~1.3% continuous regime.
Right context (NVIDIA "configurable lookahead"): one knob — att_context_size = [70, R] set at load via encoder.set_default_att_context_size([70, R]). R is the number of future chunks each conformer layer may attend to; the multi-context-trained checkpoint supports R ∈ {0, 1, 6, 13} → ≈ {80, 160, 560, 1120} ms latency (we run R=1). The silence padding at finalize is not a separate setting — it is mechanically (R+1)×shift_frames, the trailing frames the encoder must see to flush the last real frame at that R.
